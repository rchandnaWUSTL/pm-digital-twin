# Skill: Insights One-Pager

## Purpose
Take the unified Insights JSON and produce a formatted markdown one-pager suitable for sharing with the team or stakeholders.

---

## Input

Path to the Insights JSON file (`/context/insights/{customer}-{date}.json`).

---

## Output

Formatted markdown file.

---

## Output Format

```markdown
# Insights — {Customer Name} ({Date})

## Key Themes
{For each theme, ordered by frequency:}
* **{Theme title}** ({frequency} mentions, {N} customers)
   * {evidence quote 1}
   * {evidence quote 2}

## Open Asks
{For each ask:}
* **{Ask description}** — first raised {date}, mentioned {N} times
   * Evidence: {quote}
   * Jira: {[PROJECT_KEY]-XXX or "⚠️ No Jira issue"}

## Pain Points
{For each pain point:}
* **{Description}** (severity: {high/medium/low})
   * Workaround: {workaround or "None identified"}

## Tool Signals
{For each tool:}
* **{Tool name}** — {context} ({sentiment})

## Jira Summary
* Matched to existing issues: {N}
* Unmatched (new issue candidates): {N}
* Flagged for review: {N}

---
Generated by PM Intelligence System on {date}
```

---

## Instructions

1. Read the Insights JSON file
2. Format each section according to the template above
3. Use `*` for top-level bullets, `   *` for sub-bullets (consistent with field notes format)
4. Include all data — do not summarize or omit
5. Write to `/context/insights/{customer}-{date}.md`

---

## Progress Signals

```

   ✓ One-pager written to /context/insights/{customer}-{date}.md

```
