# Skill: Signal Brief

## Purpose
Generate a lightweight signal document for topics that don't yet have sufficient evidence for a full PRD (fewer than 2 customers or fewer than 3 mentions). Captures what we know so far and flags gaps.

---

## Input

- Topic/keyword
- Insights JSON from the PRD pipeline merge step

---

## Output

Formatted markdown file.

---

## Signal Source Determination

`signal-brief` determines the appropriate signal subdirectory based on evidence sources in the Insights JSON:

**Routing logic**:
1. **Field signals** — If `insights_json.customers_mentioned > 0` (customer-sourced evidence)
2. **Engineering signals** — If `insights_json.related_docs` contains engineering docs or RFCs (internal R&D exploration)
3. **Market signals** — Default (competitive intelligence, general research)

**Output path**: `/context/product-context/{signal_subdir}/{topic-slug}-signal-{YYYY-MM-DD}.md`

---

## Output Format

Frontmatter should include:
```yaml
---
project_tag: "{Topic}"
one_line_description: "Signal brief for {topic} — insufficient evidence for PRD"
doc_type: signal
signal_source: field | engineering | market  # Based on evidence sources (see Signal Source Determination)
date_ingested: YYYY-MM-DD
related_customers: []  # Populated from Insights JSON
related_jira: []       # Populated from Insights JSON
related_docs: []       # Populated from Insights JSON
---
```

Markdown body structure:
```markdown
# Signal Brief — {Topic}

**Generated:** {date}
**Evidence level:** Insufficient for PRD ({N} customers, {M} total mentions)

## What We're Hearing

{Summary of the signal in 2-3 sentences}

## Evidence

{For each piece of evidence with wikilinks to customer notes:}
* **[[Customer]]** ([[customer-YYYY-MM-DD]]): "{quote or paraphrase}"

## Customers Mentioning This
{List of customers with dates}

## Gaps
* {What additional evidence would strengthen this signal}
* {Which customer segments haven't been asked about this}

## Recommended Next Steps
* {Specific actions to gather more evidence}

---
Generated by PM Intelligence System
```

---

## Instructions

1. Read the Insights JSON
2. Determine signal source (field/engineering/market) using the routing logic above
3. Summarize what evidence exists for this topic
4. Be honest about gaps — this brief exists because evidence is insufficient
5. Suggest specific next steps to gather more signal
6. Add wikilinks: Link customer names (`[[Customer]]`) and customer notes (`[[customer-date]]`) in the Evidence section
7. Generate frontmatter with appropriate `signal_source` value
8. Write to `/context/product-context/{signal_subdir}/{topic-slug}-signal-{YYYY-MM-DD}.md`
